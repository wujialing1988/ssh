My97日期控件

扩展类：Ext.form.field.Text

扩展方式：extend

类名：Ext.form.field.My97DateTimePicker

xtype：dateTimePicker

简介：

My97日期控件是在原生的my97的基础上进行了封装简化和扩展的一个组件，将原有的复杂配置简化，并根据常用的业务需求对时间限制做了调整

扩展功能：

1.设置为必填项后会把当前时间设置默认时间。

2.支持id和时间格式参数来限制最大最小时间

3.可以配置选择时间后的回调函数

用法：

引入控件JS文件

<script type="text/javascript" src="../../scripts/extjs/ux/ExtjsExtend.js"></script>
<script type="text/javascript" src="../../scripts/common/my97DatePicker/WdatePicker.js"></script>

参数表：

dateFmt 默认：yyyy-MM-dd 用于定义日期时间的格式，如yyyy-MM-dd HH:mm:ss，表示年月日时分秒
maxDate 默认：2099-12-31 23:59:59 不得大于（晚于）控件的id，或者不得大于的时间范围如maxDate:'2016-01-11'  注意：考虑到减少参数项，所以再定义id关联控制限制条件这种方式的时候，不能使用纯数字作为id名称，比如配置一个关联控件的id:'123_1'
minDate 默认：1900-01-01 00:00:00 不得小于（早于）控件的id，或者不得小于的时间范围如minDate:'2016-01-11'  注意：同maxDate
changedCallback 当选择时间完成之后，触发的一个方法，如changedCallback:alertCallback，选择完成之后触发一个alertCallback的方法

示例1：根据id限制最大和最小时间

示例说明：

示例源码：查看

具体属性设置请参考以下代码

	   {
			fieldLabel : '开始时间',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd HH:mm:ss',//时间格式
			id:'purchaseDate',//id
			maxDate:'startRunDate'//不得大于（晚于）控件的id
	   }
	   {
			fieldLabel : '结束时间',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd HH:mm:ss',
			id:'startRunDate',
			minDate:'purchaseDate'//不得小于（早于）控件的id
	   }

示例2：根据时间条件限制最大和最小时间

示例说明：

示例源码：查看

具体属性设置请参考以下代码

	    {
			fieldLabel : '不早于时间',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd',
			maxDate : '2016-01-11'//不得大于（晚于）2016-01-11
		}
		{
			fieldLabel : '不晚于时间',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd HH:mm:ss',
			minDate : '2016-01-11'//不得小于（早于）2016-01-11
		}
		{
			fieldLabel : '同时满足',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd HH:mm:ss',
			minDate : '2016-01-11',//不得小于（早于）2016-01-11
			maxDate : '2016-01-22'//并且不得大于（晚于）2016-01-22
		}

示例3：选择日期后回调

示例说明：

示例源码：查看

具体属性设置请参考以下代码

	    {
			fieldLabel : '选择后回调',
			xtype : "dateTimePicker",
			dateFmt : 'yyyy-MM-dd',
			allowBlank : false,//设备必填项，默认当前时间
			changedCallback:alertCallback//回调函数方法alertCallback
		}
